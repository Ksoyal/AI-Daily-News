# 02 · 项目架构总览

## 一句话概述

> 每天早上 8:00 自动从 9 个 RSS 源抓新闻 → AI 生成日报 → 存到 Notion → 手机推送。

## 数据流图

```
┌─────────────┐
│  9个 RSS 源  │  互联网上的新闻订阅源
│  (XML/Atom)  │
└──────┬──────┘
       │ HTTP GET (30s 超时, User-Agent 伪装)
       ▼
┌─────────────┐
│  fetcher.py │  抓取 + 过滤 + 排序
│              │  输出: list[dict] 每条新闻 4 个字段
└──────┬──────┘
       │ 100 条新闻
       ▼
┌──────────────┐
│ summarizer.py│  构建 Prompt → 调 OpenRouter API
│              │  输出: {headline, tags, content}
└──────┬───────┘
       │ dict 结构
       ▼
┌──────────────┐
│ publisher.py │  Markdown → Notion Blocks
│              │  POST 到 Notion API 创建页面
└──────┬───────┘
       │
       ▼
┌──────────────┐
│   main.py    │  串联 + 手机推送
│              │  异常时也推送失败通知
└──────────────┘
```

## 各模块的数据契约

模块间通过**函数返回值的格式**来约定接口，没有复杂的类或继承。

```
fetcher.py
  fetch_news() → [
      {"title": "新闻标题", "link": "https://...", 
       "source": "36氪",   "published": "2026-06-04T08:00:00+00:00"},
      ...
  ]

summarizer.py
  generate_report(news_list) → {
      "headline": "特朗普取消对伊朗新攻击",
      "tags":     ["地缘政治", "AI监管", "资本市场"],
      "content":  "## 🌟 今日要闻\n...(Markdown 正文)"
  }

publisher.py
  push_to_notion(report) → None（异常则 raise）
```

## 为什么这样设计？

- **管道架构**（Pipeline）：每个模块只做一件事，上一个的输出是下一个的输入
- **无状态**：每个函数独立，不依赖外部存储，输入决定输出
- **异常隔离**：RSS 抓取一个源失败不影响其他源；AI 调用失败则整体报错（因为没法继续）
- **配置外部化**：所有参数在 `config.py`，改配置不改代码

## 关键设计模式

| 模式 | 在哪里 | 解决什么问题 |
|------|--------|------------|
| 单源异常隔离 | fetcher.py | 9 个源里 1 个挂了，剩下 8 个照常 |
| 幂等性检查 | publisher.py | 同一天跑两次不会创建两条 Notion 页面 |
| 指数退避重试 | publisher.py | 429 限流等 1s/2s/4s 再试 |
| Fallback URL | fetcher.py | NYT 直连被挡就换 RSSHub 镜像 |
| Token 预算 | summarizer.py | 按源轮询截断，保证每个源都有代表 |
