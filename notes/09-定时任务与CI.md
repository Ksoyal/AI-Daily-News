# 09 · 定时任务与 CI/CD

> GitHub Actions 是 GitHub 提供的免费自动化服务。我们用它每天自动运行 Python 脚本。

## 工作流文件

`.github/workflows/` 下的 `.yml` 文件被 GitHub 自动识别和执行。

### 主工作流 `daily_run.yml`

```yaml
name: AI Daily News              # 在 Actions 面板显示的名字

on:                               # 触发条件
  schedule:                       # 定时触发
    - cron: "0 0 * * *"           # UTC 0:00 = 北京时间 8:00
  workflow_dispatch:              # 手动触发（网页按钮）
  repository_dispatch:            # API 触发（trigger.py 用）
    types: [daily_trigger]

jobs:
  run:                            # 一个"运行"任务
    runs-on: ubuntu-latest        # 用 GitHub 的 Ubuntu 虚拟机
    timeout-minutes: 15           # 15 分钟超时自动杀进程
    
    steps:                        # 步骤按序号执行
      - name: Checkout            # ① 拉取代码
        uses: actions/checkout@v4
        
      - name: Setup Python        # ② 安装 Python 3.10
        uses: actions/setup-python@v5
        with:
          python-version: "3.10"
          
      - name: Install deps        # ③ pip 安装依赖
        run: pip install -r requirements.txt
        
      - name: Run pipeline        # ④ 执行主脚本
        env:                       # 注入 Secret 为环境变量
          OPENROUTER_API_KEY: ${{ secrets.OPENROUTER_API_KEY }}
          NOTION_TOKEN: ${{ secrets.NOTION_TOKEN }}
          NOTION_DATABASE_ID: ${{ secrets.NOTION_DATABASE_ID }}
          PUSH_KEY: ${{ secrets.PUSH_KEY }}
        run: python main.py
```

### 守门员工作流 `precise_trigger.yml`

```yaml
name: Precise Trigger

on:
  schedule:
    - cron: "*/15 * * * *"         # 每 15 分钟触发一次

jobs:
  gate:
    runs-on: ubuntu-latest
    timeout-minutes: 2
    steps:
      - name: Check time
        run: |
          HOUR=$(date -u +%H)      # 当前 UTC 小时
          MINUTE=$(date -u +%M)    # 当前 UTC 分钟
          if [ "$HOUR" = "00" ] && [ "$MINUTE" -lt "15" ]; then
            echo "fire=true" >> $GITHUB_OUTPUT
          fi
      
      - name: Trigger
        if: steps.check.outputs.fire == 'true'
        run: gh workflow run daily_run.yml
```

**原理**: GitHub 的每日 cron 不保证准时，但 15 分钟 cron 很可靠。这个工作流每 15 分钟醒一次 → 检查是不是 8:00 窗口 → 是就触发主流水线。

## Secrets 管理

在 GitHub 仓库 Settings → Secrets and variables → Actions 中设置：

| Secret | 值 |
|--------|-----|
| `OPENROUTER_API_KEY` | `sk-or-v1-xxxx` |
| `WORKFLOW_PAT` | `ghp_xxxx`（用于 trigger） |
| `NOTION_TOKEN` | `ntn_xxxx` |
| `NOTION_DATABASE_ID` | 32 位 hex |
| `PUSH_KEY` | Server酱 SendKey |

## cron 表达式速查

```
┌─ 分钟 (0-59)
│  ┌─ 小时 (0-23, UTC)
│  │  ┌─ 日 (1-31)
│  │  │  ┌─ 月 (1-12)
│  │  │  │  ┌─ 星期 (0-6, 0=周日)
│  │  │  │  │
*  *  *  *  *
```

示例:
- `0 0 * * *` — 每天 UTC 0:00
- `*/15 * * * *` — 每 15 分钟
- `0 8 * * 1-5` — 工作日 UTC 8:00
